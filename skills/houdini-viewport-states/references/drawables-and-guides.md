## Overview

Guide geometry is 3D "UI" geometry that only appears while your state is active. It's separate
from the "real" geometry generated by the SOP network. Examples: a brush radius ring, a wind
direction arrow, a preview of alignment.

Houdini provides several drawable classes for displaying guides. The geometry source can be
a `hou.Geometry` object or a `hou.drawablePrimitive` value.

Key rules:
- Store drawable references on `self` — Python garbage collection will delete unreferenced drawables
- Drawables render in **world space** — compensate for parent transforms if needed
- Drawables don't appear instantly — they show on next viewer redraw (tumble, or force with
  `self.scene_viewer.curViewport().draw()`)
- Prefer transforming drawables over recreating them for animation/tracking

## Drawable Types

| Class | Use Case |
|-------|----------|
| `hou.SimpleDrawable` | General-purpose guide geometry (wireframe by default) |
| `hou.GeometryDrawable` | Advanced rendering with fine-grained display options |
| `hou.TextDrawable` | Text labels in the viewport |
| `hou.GadgetDrawable` | Pickable/locatable geometry (lightweight custom handles) |

## SimpleDrawable

The most common drawable for guide geometry. Defaults to wireframe display.

### Creating from Geometry
```python
def onEnter(self, kwargs):
    node = kwargs["node"]
    # From a SOP node inside the asset
    geo = node.node("guide_output").geometry()
    
    self._guide = hou.SimpleDrawable(
        self.scene_viewer, geo,
        self.state_name + "_guide"  # unique name
    )
    self._guide.enable(True)
    self._guide.show(True)
```

### Creating from a Primitive Type
```python
self._sphere = hou.SimpleDrawable(
    self.scene_viewer,
    hou.drawablePrimitive.Sphere,
    self.state_name + "_sphere"
)
self._sphere.setWireframeColor(hou.Color(1, 0, 0))  # Red wireframe
self._sphere.enable(True)
self._sphere.show(True)
```

### Available Primitive Types
`hou.drawablePrimitive`: `Sphere`, `Circle`, `Line`, `Box`, `Plane`, `Cone`, `Cylinder`, etc.

### Display Modes
```python
self._guide.setDisplayMode(hou.drawableDisplayMode.WireframeMode)   # default
self._guide.setDisplayMode(hou.drawableDisplayMode.CurrentViewportMode)
self._guide.setWireframeColor(hou.Color(0.5, 0.8, 1.0))
```

### Show/Hide Pattern
```python
def onInterrupt(self, kwargs):
    self._guide.show(False)

def onResume(self, kwargs):
    self._guide.show(True)
```

### Accessing/Updating Geometry
The drawable keeps a live reference to the Geometry object. If the underlying geometry changes
(e.g., driven by parameters on your asset), the drawable updates automatically:
```python
geo = self._guide.geometry()  # read-only reference
```

## GeometryDrawable

Advanced drawable with more rendering control than SimpleDrawable. Does NOT default to 
wireframe — you configure display options explicitly.

```python
self._drawable = hou.GeometryDrawable(
    self.scene_viewer,
    hou.drawableGeometryType.Face,  # or .Line, .Point
    self.state_name + "_drawable"
)
self._drawable.setGeometry(my_geometry)
self._drawable.setParams({
    "color1": (1.0, 0.0, 0.0, 1.0),   # RGBA
    "fade_factor": 0.5,
    "line_width": 2.0,
    "style": hou.drawableGeometryLineStyle.Plain,
})
self._drawable.show(True)
```

### GeometryDrawable Types
- `hou.drawableGeometryType.Face` — filled polygons
- `hou.drawableGeometryType.Line` — edges/wireframe
- `hou.drawableGeometryType.Point` — point markers

### Drawing in onDraw
GeometryDrawable (and GadgetDrawable) require explicit drawing in `onDraw`:
```python
def onDraw(self, kwargs):
    handle = kwargs["draw_handle"]
    self._drawable.draw(handle)
```

## TextDrawable

For rendering text labels in the viewport.

```python
self._text = hou.TextDrawable(
    self.scene_viewer,
    self.state_name + "_text"
)
self._text.setParams({
    "text": "Hello World",
    "translate": hou.Vector3(0, 1, 0),  # 3D position
    "color1": (1.0, 1.0, 1.0, 1.0),
    "highlight_mode": hou.drawableHighlightMode.MatteOverGeo,
    "font_size": 14,
})
self._text.show(True)
```

### Drawing Text
```python
def onDraw(self, kwargs):
    handle = kwargs["draw_handle"]
    self._text.draw(handle)
```

## Gadgets

Gadgets are specialized drawables with built-in picking and locating support. They provide
a lightweight alternative to full Python handles when you need interactive geometry that 
responds to mouse hover and clicks.

### When to Use Gadgets vs Drawables vs Handles

| Feature | SimpleDrawable | GeometryDrawable | GadgetDrawable | Python Handle |
|---------|---------------|-------------------|----------------|---------------|
| Display geometry | ✅ | ✅ | ✅ | ✅ |
| Pick/locate components | ❌ | ❌ | ✅ | ✅ |
| Parameter binding | ❌ | ❌ | ❌ | ✅ |
| Reusable across states | ❌ | ❌ | ❌ | ✅ |
| Complexity | Low | Medium | Medium | High |

Use gadgets when you need to know what component the user is hovering over or clicking on,
but don't need the full power of a Python handle.

### Registering Gadgets
Gadgets are registered on the template during state registration:
```python
def createViewerStateTemplate():
    template = hou.ViewerStateTemplate(...)
    template.bindFactory(State)
    
    # Register gadgets by geometry type
    template.bindGadget(hou.drawableGeometryType.Line, "line_gadget", label="Line")
    template.bindGadget(hou.drawableGeometryType.Face, "face_gadget", label="Face")
    template.bindGadget(hou.drawableGeometryType.Point, "point_gadget", label="Point")
    
    return template
```

### Using Gadgets in the State
Gadget instances are auto-created by Houdini and assigned to `self.state_gadgets`:
```python
def onEnter(self, kwargs):
    node = kwargs["node"]
    geo = node.geometry()
    
    self.face_gadget = self.state_gadgets["face_gadget"]
    self.face_gadget.setGeometry(geo)
    self.face_gadget.setParams({
        "draw_color": [0, 0.3, 0, 1],
        "locate_color": [1, 0, 0, 1],
        "pick_color": [1, 1, 0, 1]
    })
    self.face_gadget.show(True)

def onMouseEvent(self, kwargs):
    # self.state_context is auto-created by Houdini
    gadget_name = self.state_context.gadget()
    if gadget_name == "face_gadget":
        component_id = self.state_context.component1()
        # component_id is the prim/point/edge index under the cursor
    return True

def onDraw(self, kwargs):
    handle = kwargs["draw_handle"]
    self.face_gadget.draw(handle)
```

### Gadgets with ViewerStateDragger
Gadgets work with `hou.ViewerStateDragger` for dragging interactions. See `patterns.md`
for dragger usage patterns.

## Generating Guide Geometry

### Programmatic (SOP Verbs)
Build geometry from scratch using SOP verbs:
```python
sops = hou.sopNodeTypeCategory()
box_verb = sops.nodeVerb("box")
box_verb.setParms({"scale": 0.25})

geo = hou.Geometry()
temp = hou.Geometry()

for pos in [(0, 0, 0), (1, 0, 0), (0, 1, 0)]:
    box_verb.setParms({"t": hou.Vector3(*pos)})
    box_verb.execute(temp, [])
    geo.merge(temp)  # Verbs overwrite, so merge into main geo
```
Note: `verb.execute()` overwrites the target geometry. Always execute into a temp buffer
and merge into your main geometry.

### From Asset Internals
Cook a SOP node inside your asset to generate guides:
```python
def onEnter(self, kwargs):
    node = kwargs["node"]
    geo = node.node("guide_output").geometry()  # live reference — auto-updates
    self._guide = hou.SimpleDrawable(self.scene_viewer, geo, "guide")
    self._guide.enable(True)
    self._guide.show(True)
```
This is powerful because the guide network can be parameter-driven without scripting.

### From drawablePrimitive
For simple shapes, use built-in primitives:
```python
self._cursor = hou.SimpleDrawable(
    self.scene_viewer,
    hou.drawablePrimitive.Circle,
    "cursor"
)
```

## Transforms

Transform drawables instead of recreating them — much more efficient for animation/tracking:

```python
# Build a transform matrix
xform = hou.hmath.buildTranslate(position)
xform *= hou.hmath.buildRotate(rx, ry, rz)
xform *= hou.hmath.buildScale(sx, sy, sz)

self._guide.setTransform(xform)
```

Each piece of guide geometry that needs independent transforms must be in separate Drawable objects.

## Drawable Lifecycle & Performance

1. **Create once** (in `onEnter`), reuse across events
2. **Transform, don't recreate** — changing transforms is far cheaper than rebuilding geometry
3. **Cache geometry references** — for intersection, hold one reference rather than re-fetching
4. **Force redraw** when needed: `self.scene_viewer.curViewport().draw()`
5. **Guide geometry should be closed polygon meshes** — other types may not render correctly
6. **`enable(True)` vs `show(True)`**: Both are needed. `enable` registers the drawable with 
   the viewport; `show` controls visibility. Typically set both in `onEnter`.

## Compensating for Parent Transforms

Drawables render in world space. If your SOP is inside a transformed OBJ:
```python
import viewerstate.utils as su

def _world_xform(self, node):
    obj_node = su.ancestorObject(node)
    if obj_node:
        return obj_node.worldTransform()
    return hou.Matrix4(1)

def onEnter(self, kwargs):
    node = kwargs["node"]
    world = self._world_xform(node)
    self._guide.setTransform(world)
```
